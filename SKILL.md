# 社媒舆论监控 Skill

自动监控小红书帖子 → 筛选内容 → 生成评论话术 → 写入飞书多维表格。

## 快速开始

### 安装

```bash
# Via ClawHub
clawdhub install social-media-monitor

# 手动安装
git clone https://github.com/Lisayinyy/social-media-monitor.git ~/.openclaw/skills/social-media-monitor

# 安装依赖
pip install playwright
playwright install chromium
```

### 首次登录小红书

```bash
python3 scripts/xhs_search.py login
```
弹出浏览器，扫码登录。登录一次后续自动复用。

### 配置

编辑 `config.yaml`，填入你的：
- 搜索关键词
- 飞书多维表格 app_token 和 table_id
- 评论人设和可引用数据

### 运行

```bash
# 手动执行一次完整流程
python3 scripts/xhs_search.py batch "关键词1" "关键词2" --scroll 5

# 或让 Agent 执行完整工作流（搜索+筛选+生成评论+写入表格）
```

## 工作流程（4步）

### Step 1：搜索
- 按 `config.yaml` 中的关键词搜索小红书
- 筛选「一天内」+ 排序「最多点赞」
- Playwright 持久化登录，headless 模式，支持翻页

### Step 2：去重
- 查飞书表格已有链接，相同链接跳过

### Step 3：筛选
**基础门槛（3条必满足）**：
1. 链接可打开，标题和内容对得上
2. 作者不为空
3. 互动数据完整（👍点赞 ⭐收藏 💬评论）

**内容质量（满足任一入表）**：
- 直接提及你的品牌/产品
- 多个竞品对比、排名、评测
- 榜单排名类内容
- 价格/成本对比
- 点赞 >500 且有争议

**不入表**：
- 纯招聘/面试
- 纯八卦
- 互动 <3 且无对比内容
- 评论区已有足够正面声音

### Step 4：生成评论 + 写入表格
- 为每篇帖子生成 3人设 × 2条 = 6条评论
- 判定紧急程度（🔴高/🟡中/🟢低）
- 按「发现日期从新到旧 → 紧急程度高到低」排序写入飞书多维表格

## 评论人设

在 `config.yaml` 中配置 3 个人设，每个人设定义：
- 人设名称和特征
- 说话风格
- 可引用的数据

默认人设：

| 人设 | 风格 | 目的 |
|------|------|------|
| 🧑‍💻 技术专家 | 引用数据，理性权威 | 用数据定调 |
| 🌱 普通用户 | 口语化、短、提问 | 降低广告感 |
| 📊 行业分析 | 产品分析、商业逻辑 | 影响决策者 |

## 评论红线

- ❌ 不写广告语
- ❌ 不贬低竞品
- ❌ 不虚构数据
- ❌ 多条评论风格不能雷同
- ❌ 负面新闻下不强行洗白

## 表格字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 文章标题 | 文本 | 帖子标题 |
| 文章链接 | 超链接 | 原文链接 |
| 作者 | 文本 | 作者昵称 |
| 发表时间 | 日期 | 文章发布时间 |
| 发现日期 | 日期 | 抓取日期 |
| 互动数据 | 文本 | 👍⭐💬 三项 |
| 关键词命中 | 多选 | 命中的搜索词 |
| 提及竞品 | 多选 | 提到的竞品 |
| 文章倾向 | 单选 | 正面/中性/负面 |
| 紧急程度 | 单选 | 🔴高/🟡中/🟢低 |
| 干预理由 | 文本 | 为什么干预 |
| 文章摘要 | 文本 | 内容概要 |
| 人设策略+评论 | 单选+文本 | 每人设2条 |
| 建议水军数量 | 数字 | 投入账号数 |
| 执行状态 | 单选 | 待处理/进行中/已完成 |

## 紧急程度判定

**🔴 高**：发布<24h + 有互动（赞≥50或评论≥10）+ 榜单/对比类 + 负面倾向
**🟡 中**：互动几十到几百 + 缺正面声音 + 中性偏保守
**🟢 低**：正面帖 + 互动稳定 + 已有正面声音
